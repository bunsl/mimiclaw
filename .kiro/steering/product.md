# 产品概述

MimiClaw 是运行在 5 美元 ESP32-S3 芯片上的 AI 助手。纯 C 语言裸机实现（无 Linux、无 Node.js），将微控制器变成可通过 Telegram 或 WebSocket 访问的个人 AI 代理。

## 核心功能

- **对话式 AI 代理**：支持工具调用的 ReAct 模式
- **多 LLM 提供商**：支持 Anthropic Claude 和 OpenAI GPT，运行时可切换
- **本地记忆持久化**：SOUL（个性）、USER（用户信息）、MEMORY（长期记忆）、每日笔记
- **会话历史管理**：每个聊天的历史以 JSONL 格式存储
- **网络搜索集成**：支持 Tavily（推荐）或 Brave Search API
- **Cron 调度器**：AI 可自主创建周期性或一次性任务
- **心跳服务**：定期检查任务列表并自主执行
- **WebSocket 网关**：局域网访问接口（端口 18789）
- **OTA 固件更新**：通过 WiFi 无线更新
- **串口 CLI**：配置和调试命令行界面

## 目标硬件

- **芯片**：ESP32-S3（Xtensa LX7 双核 @ 240 MHz）
- **Flash**：16 MB（用于固件和文件存储）
- **PSRAM**：8 MB（用于大缓冲区）
- **供电**：USB Type-C（0.5W，可 24/7 运行）
- **网络**：WiFi 2.4GHz（必需）
- **参考板**：小智 AI 开发板（约 10 美元）

## 用户交互方式

### 主要接口
- **Telegram Bot**：主要交互方式，通过 Telegram 应用发送消息
- **WebSocket 客户端**：局域网访问（端口 18789），支持最多 4 个并发连接
- **串口 CLI**：通过 USB UART 端口进行配置和调试

### 交互流程
1. 用户通过 Telegram 发送消息
2. ESP32-S3 通过 WiFi 接收消息
3. 代理循环处理：LLM 推理 → 工具调用 → 读取记忆
4. 响应通过相同通道返回给用户

## 设计理念

### 极简主义
- 纯 C 语言实现，无脚本语言依赖
- FreeRTOS 实时操作系统，无 Linux 开销
- 直接 HTTP/HTTPS 调用，无中间层框架

### 本地优先
- 所有数据存储在设备 Flash（SPIFFS）
- 会话历史、记忆、配置均在本地
- 人类可读的纯文本格式（Markdown、JSON、JSONL）

### 主动性
- 心跳服务定期检查任务列表
- AI 可自主创建和执行定时任务
- 无需用户触发即可完成周期性工作

### 可扩展性
- 工具注册表模式，易于添加新工具
- 技能加载器支持动态加载技能定义
- Cron 调度器支持复杂的任务编排

### 资源高效
- 双核任务分配：Core 0 处理 I/O，Core 1 处理 AI
- 智能内存管理：小缓冲区用 SRAM，大缓冲区用 PSRAM
- 会话历史环形缓冲区，自动限制大小

## 典型使用场景

- **个人助手**：日程提醒、信息查询、任务管理
- **智能家居控制**：通过 GPIO 工具控制外设
- **信息聚合**：定期搜索和总结特定主题
- **自动化任务**：通过 Cron 执行周期性操作
- **学习和实验**：嵌入式 AI 代理的教学平台
